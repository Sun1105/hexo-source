<h2>Spring トランザクション動作（3パターン比較）</h2>

<!-- ================= 日本语 ================= -->
<div class="section">
<div class="title">🇯🇵 日本語説明</div>

<div class="flow-row">

<!-- Case 1 -->
<div class="flow-box safe">
<b>① Serviceのみ@Transactional</b>

<div class="arrow">↓</div>
例外発生（RuntimeException）

<div class="arrow">↓</div>
catch（例外を握りつぶす）

<div class="arrow">↓</div>
Spring：正常終了と判断

<div class="arrow">↓</div>
<span class="green">commit</span>

</div>

<!-- Case 2 -->
<div class="flow-box error">
<b>② Service + Component（REQUIRED）</b>

<div class="arrow">↓</div>
内部で例外発生

<div class="arrow">↓</div>
<span class="red">setRollbackOnly</span>

<div class="arrow">↓</div>
catch（例外を握りつぶす）

<div class="arrow">↓</div>
commit実行

<div class="arrow">↓</div>
<span class="red">rollback-only と衝突</span>

<div class="arrow">↓</div>
<span class="red">UnexpectedRollbackException</span>

</div>

<!-- Case 3 -->
<div class="flow-box isolated">
<b>③ REQUIRES_NEW</b>

<div class="arrow">↓</div>
外部Tx-A 開始

<div class="arrow">↓</div>
<span class="blue">Tx-A 一時停止</span>

<div class="arrow">↓</div>
<span class="blue">Tx-B 開始</span>

<div class="arrow">↓</div>
commit（Tx-B）

<div class="arrow">↓</div>
Tx-A 再開

<div class="arrow">↓</div>
外部rollbackしても影響なし

</div>

</div>

<br>

<b>ポイント：</b>
<ul>
<li>@Transactional により 監視ポイントが増える</li>
<li>rollback 判定は「例外伝播」に依存する</li>
<li>REQUIRES_NEW はトランザクションを完全分離する</li>
</ul>

</div>

<!-- ================= 中文 ================= -->
<div class="section">
<div class="title">🇨🇳 中文说明</div>

<div class="flow-row">

<!-- Case 1 -->
<div class="flow-box safe">
<b>① 只有Service有@Transactional</b>

<div class="arrow">↓</div>
发生异常

<div class="arrow">↓</div>
catch（异常被吞）

<div class="arrow">↓</div>
Spring认为正常

<div class="arrow">↓</div>
<span class="green">commit（正常提交）</span>

</div>

<!-- Case 2 -->
<div class="flow-box error">
<b>② Service + Component（默认REQUIRED）</b>

<div class="arrow">↓</div>
发生异常

<div class="arrow">↓</div>
<span class="red">内部事务检测 → rollback-only</span>

<div class="arrow">↓</div>
catch（异常被吞）

<div class="arrow">↓</div>
commit

<div class="arrow">↓</div>
<span class="red">rollback-only 与 commit 冲突</span>

<div class="arrow">↓</div>
<span class="red">UnexpectedRollbackException</span>

</div>

<!-- Case 3 -->
<div class="flow-box isolated">
<b>③ 使用 REQUIRES_NEW</b>

<div class="arrow">↓</div>
主事务（Tx-A）

<div class="arrow">↓</div>
<span class="blue">暂停 Tx-A</span>

<div class="arrow">↓</div>
<span class="blue">开启新事务 Tx-B</span>

<div class="arrow">↓</div>
Tx-B 提交

<div class="arrow">↓</div>
恢复 Tx-A

<div class="arrow">↓</div>
主事务回滚也不影响 Tx-B

</div>

</div>

<br>

<b>核心结论：</b>
<ul>
<li>@Transactional = 增加“异常监听点”</li>
<li>是否回滚取决于：异常是否逃出事务边界</li>
<li>REQUIRES_NEW = 完全独立事务</li>
</ul>

</div>
