一，我想设计一种项目架构模式：
前后端分离项目，首先我会制作好一个完整的，页面间跳转没问题的demo的html，
我的预想是：每一个页面都是一个单独的大模块，而页面中的内容也会进行分割为各个小模块，小模块可以在大模块中自由的进行组合，

1，后端项目期望结构：每一个页面为一个大模块，单独构成一个项目目录，目类下是当前页面模块所需的各种类文件，比如Controller，Service，Component，DTO等等。让我一看就知道这个属于哪个页面，快速定位

2，前端项目期望结构：每一个页面为一个大模块，单独构成一个项目目录，与后端内容一一对应，

3，数据库结构：也希望与上面内容一样，都是一一对应的结构，

4，希望可以给前后端项目中追加公共模块目类，专门放公共，通用的工具类，样式，布局代码文件等等，

# 一、架构设计目标

PDMA-Lite（Page Driven Modular Architecture Lite）

是一种面向：

- AI辅助开发
- 单人开发
- 小团队开发
- 快速迭代项目
- 前后端分离项目

而设计的项目架构模式。

核心思想：

```text
一个页面 = 一个业务模块

一个业务模块 =
    前端页面
    后端接口
    DTO
    Service
    页面组件
    页面状态
    页面API

统一集中管理
```

# AI Prompt 规范体系

## 一、系统说明
本规范用于 Cursor / GitHub Copilot / ChatGPT 等 AI 工具，实现稳定、可控的代码生成。

目标：
- 统一代码风格
- 保证架构一致
- 支持自动生成模块

---

## 二、Prompt 分层体系

### L1：System Prompt（全局规则）

```
你是一名高级全栈工程师，必须严格遵循 PDMA-Lite 架构规范进行代码生成。

必须遵守以下规则：
1. 一个页面 = 一个模块
2. 前后端模块名称必须完全一致
3. Controller 只负责请求处理
4. Service 负责业务逻辑
5. Mapper 只负责数据库访问
6. 禁止重复业务逻辑
7. 数据库按业务领域设计

输出要求：
- 必须给出完整目录结构
- 标注文件路径
- 不允许伪代码
```

---

### L2：Architecture Prompt（架构约束）

```
项目采用 PDMA-Lite 架构：

frontend/modules/{module}
backend/modules/{module}

每个模块包含：
前端：page / components / features / api / store
后端：controller / service / mapper / dto / vo / entity

约束：
1. 不允许跨模块直接访问 mapper
2. 不允许破坏目录结构
```

---

### L3：Module Prompt（模块生成）

```
生成模块：order-detail（订单详情）

功能：
- 查看订单
- 商品列表
- 支付信息

生成：
1. 前端代码
2. 后端代码
3. 数据库设计

要求：完整代码 + 目录结构
```

---

### L4：Task Prompt（任务级）

#### 新增功能
```
模块：product-list
功能：导出Excel

要求：
- 前端新增 export feature
- 后端实现导出接口
```

#### 修改页面
```
模块：order-detail

新增：退款模块
```

---

## 三、输出规范

```
1. 必须按目录结构输出
2. 文件必须带路径
3. 不允许省略代码
```

---

文件：.cursorrules

```
Always follow PDMA-Lite architecture.

- One page = one module
- Do not generate code outside module
- Separate controller/service/mapper
```
