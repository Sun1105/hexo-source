  <div class="wrapper">
    <h1 class="title">业务逻辑 vs 技术实现 对照图</h1>
    <div class="subtitle">以“电商下单处理”为例</div>

    <div class="note">
      核心理解：业务逻辑回答“为什么这样做、做什么”，技术实现回答“怎么做”
    </div>

    <div class="grid">
      <div class="head blank"></div>
      <div class="head biz">业务逻辑（做什么 / 为什么）</div>
      <div class="head mid">对应关系</div>
      <div class="head tech">技术实现（怎么做）</div>

      <div class="cell step">1<br>提交订单</div>
      <div class="cell biz">
        <div class="section-title">业务描述</div>
        用户在商品确认后点击“提交订单”。
        <ul>
          <li>必须先选定商品和数量</li>
          <li>必须有收货地址</li>
        </ul>
      </div>
      <div class="cell mid">发起业务请求</div>
      <div class="cell tech">
        <div class="section-title">技术实现</div>
        前端表单提交到后端接口。
        <div class="code">POST /order/submit
参数：
- userId
- items
- addressId</div>
      </div>

      <div class="cell step">2<br>校验库存</div>
      <div class="cell biz">
        <div class="section-title">业务描述</div>
        下单前必须确认商品库存是否足够。
        <ul>
          <li>库存不足则不能下单</li>
          <li>库存足够才进入下一步</li>
        </ul>
      </div>
      <div class="cell mid">业务规则判断</div>
      <div class="cell tech">
        <div class="section-title">技术实现</div>
        后端查询库存服务或数据库。
        <div class="code">if (stock < orderQty) {
    throw new BusinessException("库存不足");
}</div>
      </div>

      <div class="cell step">3<br>计算价格</div>
      <div class="cell biz">
        <div class="section-title">业务描述</div>
        根据商品单价、数量、优惠券、满减规则计算应付金额。
        <ul>
          <li>优惠规则按业务配置执行</li>
          <li>最终金额需明确展示给用户</li>
        </ul>
      </div>
      <div class="cell mid">价格规则</div>
      <div class="cell tech">
        <div class="section-title">技术实现</div>
        调用价格计算模块，组装各类金额项。
        <div class="code">total = itemTotal - couponAmount - discountAmount
payable = Math.max(total, 0)</div>
      </div>

      <div class="cell step">4<br>生成订单</div>
      <div class="cell biz">
        <div class="section-title">业务描述</div>
        将订单信息正式写入系统。
        <ul>
          <li>订单号唯一</li>
          <li>状态初始为“待支付”</li>
          <li>保存订单明细</li>
        </ul>
      </div>
      <div class="cell mid">落库处理</div>
      <div class="cell tech">
        <div class="section-title">技术实现</div>
        通过事务写入订单主表和明细表。
        <div class="code">@Transactional
orderMapper.insert(order);
orderItemMapper.batchInsert(items);</div>
      </div>

      <div class="cell step">5<br>锁定库存</div>
      <div class="cell biz">
        <div class="section-title">业务描述</div>
        订单创建后，库存应被冻结，防止超卖。
        <ul>
          <li>锁定成功才算下单成功</li>
          <li>失败时需要回滚订单</li>
        </ul>
      </div>
      <div class="cell mid">并发控制</div>
      <div class="cell tech">
        <div class="section-title">技术实现</div>
        调用库存接口或使用消息队列处理。
        <div class="code">inventoryService.lockStock(orderId, items);
// 或发送 MQ 消息异步锁定</div>
      </div>

      <div class="cell step">6<br>返回结果</div>
      <div class="cell biz">
        <div class="section-title">业务描述</div>
        返回下单是否成功，以及订单号、支付金额等信息。
        <ul>
          <li>成功：进入支付流程</li>
          <li>失败：提示具体原因</li>
        </ul>
      </div>
      <div class="cell mid">结果输出</div>
      <div class="cell tech">
        <div class="section-title">技术实现</div>
        封装统一响应对象返回前端。
        <div class="code">return ResponseEntity.ok(
    Result.success(orderNo, payableAmount)
);</div>
      </div>
    </div>

    <div class="footer">
      <div class="footer-card biz">
        <h3>业务逻辑的本质</h3>
        <p>定义系统在现实业务中“应该做什么”，以及在什么条件下做出什么判断。</p>
      </div>
      <div class="footer-card mid">
        <h3>二者的关系</h3>
        <p>业务逻辑是规则本体，技术实现是把规则落到代码、数据库和接口中的手段。</p>
      </div>
      <div class="footer-card tech">
        <h3>开发时的训练方式</h3>
        <p>先把需求翻译成业务规则，再把规则拆成代码步骤、接口调用和数据处理。</p>
      </div>
    </div>
