# 💻 新电脑文件管理初始化手册

---
**分类：** 电脑技巧
**标签：** #标准化 #初始化 #文件管理 #效率
---

## 🧭 核心逻辑：线性流转
> **原则：** 文件夹深度不超过 3 层；下载不放桌面；命名必带日期。

---

## 一、 目录架构 (根目录建议设在 D 盘)

| 文件夹名称 | 对应功能 | 管理策略 |
| :--- | :--- | :--- |
| **01_Inbox** | **收件箱** | 所有浏览器下载、临时接收的文件。**每周清空**。 |
| **02_Doing** | **进行中** | 当前正在处理的项目、学习中的课程。**完结即移走**。 |
| **03_Local** | **资料库** | 个人作品、软件配置、素材、长期参考文档。 |
| **04_Arch** | **归档库** | 按年份（如 2025、2026）存放已结束的旧项目。 |

---

## 二、 命名规范 (必执行)

使用 **[日期]_[关键词]_[版本]** 格式，确保文件自动按时间完美排序。

* **格式：** `YYYYMMDD_文件名_版本号`
* **示例：** `20260413_项目策划案_V1.docx`
* **注意：** 严禁使用“最终版”、“最最后版”，统一使用 `V1/V2/Final`。

---

## 三、 新电脑配置清单 (Checklist)

### 1. 系统设置优化
- [ ] **修改下载路径：** 将浏览器默认下载位置更改为 `D:\01_Inbox`。
- [ ] **桌面清零：** 桌面仅存放“当天待办”的快捷方式，不存放实际文件。
- [ ] **任务栏加固：** 使用“伪装法”将常用的 `.bat` 或 `.hta` 工具固定到任务栏。

### 2. 必备神器安装
- [ ] **Everything：** 实现全盘秒级搜索（解决“文件在哪”）。
- [ ] **QuickLook：** 按空格键预览文件内容（解决“文件是啥”）。

---

## 四、 定期维护计划

* **日清：** 下班/关机前，清空桌面临时文件，分类至 `01` 或 `02`。
* **周清：** 每周五处理 `01_Inbox` 中的堆积物，该删的删，该存的存。
* **季归：** 每季度将 `02_Doing` 中已完成的项目移入 `04_Arch`。

---
> **💡 笔记心法：** > 好的系统不是为了“藏”文件，而是为了在 3 秒钟内“找”到文件。

# 💻 IT 环境标准化：文件架构管理

---
**分类：环境搭建**
**标签：#SOP #文件规范 #效率提升 #Windows**
---

## 🛠️ 核心逻辑：模块化存储
> **原则：** 路径固定、版本受控、检索秒级。

---

## 一、 标准目录结构 (建议根目录：`D:\Workspace`)

| 文件夹名 | IT 定义 | 说明 |
| :--- | :--- | :--- |
| **01_Temp** | **缓冲区** | 存放 `Git Clone` 测试、下载包、临时 Log。**定期清理**。 |
| **02_Dev** | **开发中** | 活跃的项目代码、正在编辑的文档、容器挂载目录。 |
| **03_Repo** | **资源库** | 稳定的软件安装包 (Softs)、脚本工具 (Tools)、技术文档 (Docs)。 |
| **04_Safe** | **备份区** | 密钥备份、旧版本镜像、冷数据归档。 |

---

## 二、 命名与版本管理

遵循 **ISO 8601** 扩展日期格式，确保全局字符排序一致性：

* **格式：** `YYYYMMDD_[项目名]_[描述]_[版本]`
* **示例：** `20260413_NginxConf_Prod_V1.2.zip`
* **规范：** 路径避免空格和中文（尤其是脚本调用的目录），统一用下划线 `_`。

---

## 三、 IT 环境初始化 SOP

### 1. 路径重定向
- [ ] **Browser：** 修改 `Downloads` 默认路径至 `D:\01_Temp`。
- [ ] **Env Var：** 将 `D:\03_Repo\Tools` 加入系统 **Path** 环境变量（方便 Cmd 直接调用自定义脚本）。

### 2. 高效工具链
- [ ] **Everything：** 索引全盘，替代 Windows 自带搜索。
- [ ] **QuickLook：** 实现类似 macOS 的空格预览功能。
- [ ] **Terminal：** 配置 Windows Terminal 默认起始目录为 `D:\02_Dev`。

---

## 四、 自动化维护

* **日清理：** 清空 `01_Temp` 中的临时测试文件。
* **周备份：** 将 `02_Dev` 中的核心配置同步至云端或 Git 私库。
* **月归档：** 将已关闭的 Project 移动至 `04_Safe`。

---
> **💡 笔记心法：**
> 程序员的时间应花在解决 Bug 上，而不是找文件上。
